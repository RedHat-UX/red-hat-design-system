Chapter 2. Learn more about OpenShift Container Platform

Use the following sections to find content to help you learn about and better understand OpenShift Container Platform functions:

2.1. Learning and support

Find comprehensive learning resources and support options for OpenShift Container Platform. Whether you're a beginner looking to understand the basics or an experienced administrator seeking advanced configuration guidance, our extensive documentation library provides step-by-step tutorials, best practices, and troubleshooting guides. Access community forums, knowledge base articles, and official Red Hat support channels to get help when you need it most.

2.2. Architecture

Learn about the OpenShift Container Platform architecture and core components. OpenShift is built on Kubernetes and provides additional enterprise features including integrated container registry, CI/CD pipelines, monitoring, and security policies. Understanding the control plane nodes, worker nodes, etcd clusters, and networking components is essential for effective cluster management. Explore how pods, services, routes, and operators work together to create a robust container orchestration platform.

2.3. Installation

Explore the following OpenShift Container Platform installation tasks and methods. Choose from multiple installation approaches including installer-provisioned infrastructure, user-provisioned infrastructure, or assisted installer for bare metal deployments. Learn about pre-installation requirements, network configuration, storage considerations, and security settings. Whether deploying on public clouds like AWS, Azure, or Google Cloud, or on-premises infrastructure, follow detailed guides for successful cluster deployment and initial configuration.

2.4. Other cluster installer tasks

Install a cluster in a restricted network

Learn how to install OpenShift in environments with limited network access. Disconnected or air-gapped installations require special preparation including mirroring container images, configuring registry mirrors, and setting up local package repositories. This approach is common in secure environments or locations with limited internet connectivity where security policies restrict external network access.

Install a cluster in an existing network

Configure OpenShift installation to work with your existing network infrastructure. Integrate with corporate firewalls, proxy servers, DNS systems, and load balancers while maintaining compliance with organizational networking policies. Plan IP address allocation, VLAN configurations, and routing rules to ensure seamless integration with existing enterprise systems and applications.

2.5. Cluster Administrator

2.5.1. Managing and changing cluster components

Managing cluster components

Learn how to manage various cluster components and services. Monitor cluster health, manage node lifecycle, configure operator subscriptions, and maintain cluster certificates. Use the OpenShift web console, command-line tools, and APIs to perform administrative tasks including user management, resource quotas, and network policies. Implement backup strategies, security scanning, and compliance monitoring for production environments.

Changing cluster components

Understand how to safely modify cluster configurations and components. Plan maintenance windows, apply cluster updates, modify worker node configurations, and update cluster operators. Learn about rolling updates, blue-green deployments, and rollback procedures to minimize downtime during configuration changes. Test modifications in development environments before applying to production clusters.

2.6. Observe a cluster

Monitor and observe your OpenShift cluster health and performance. Utilize built-in monitoring tools including Prometheus, Grafana, and AlertManager to track metrics, set up alerts, and create custom dashboards. Monitor resource utilization, application performance, network traffic, and security events. Configure log aggregation with OpenShift Logging to centralize application and system logs for troubleshooting and compliance reporting. Set up proactive monitoring to identify issues before they impact users.

2.7. Storage activities

Configure and manage storage solutions for your OpenShift cluster. Implement persistent volume claims, storage classes, and dynamic provisioning for stateful applications. Choose from various storage backends including block storage, file systems, and object storage depending on application requirements. Configure backup and disaster recovery solutions, manage storage quotas, and optimize performance for database workloads and other data-intensive applications running in containers.

2.8. Application Site Reliability Engineer (App SRE)

Resources and tools for Site Reliability Engineers working with OpenShift applications. Focus on service level objectives, error budgets, and reliability engineering practices for containerized applications. Implement automated testing, canary deployments, and progressive delivery strategies. Use chaos engineering principles to test system resilience and failure scenarios. Design monitoring strategies that provide visibility into application performance, user experience, and business metrics while maintaining high availability and reliability standards.

2.9. Developer

Develop and deploy containerized applications with OpenShift Container Platform. Use source-to-image builds, container images, and GitOps workflows to streamline application development and deployment. Leverage OpenShift developer tools including the web console, CLI, and IDE extensions for efficient coding and debugging. Configure continuous integration and continuous deployment pipelines, manage application dependencies, and implement security best practices. Access built-in services for databases, messaging, and caching to accelerate application development.

2.10. Hosted control planes

Learn about deploying and managing hosted control planes in OpenShift. This architecture separates the control plane from worker nodes, enabling greater flexibility and cost optimization for multi-tenant environments. Understand how to provision hosted clusters, manage lifecycle operations, and configure networking and security policies. Explore benefits including improved resource utilization, simplified cluster management, and enhanced isolation between different teams and applications sharing the same infrastructure.

  4. Click Create to create the issue.

Chapter 4. About OpenShift Kubernetes Engine

As of 27 April 2020, Red Hat has decided to rename Red Hat OpenShift Container Engine to Red Hat OpenShift Kubernetes Engine to better communicate what value the product offering delivers.

Red Hat OpenShift Kubernetes Engine is a product offering from Red Hat that lets you use an enterprise class Kubernetes platform as a production platform for launching containers.

4.1. Similarities and differences

You can see the similarities and differences between OpenShift Kubernetes Engine and OpenShift Container Platform in the following comparison.

4.1.1. Core Kubernetes and container orchestration

OpenShift Kubernetes Engine offers full access to an enterprise-ready Kubernetes environment that is easy to install and offers an extensive compatibility test matrix.

4.2. Subscription limitations

OpenShift Kubernetes Engine is a subscription offering that provides OpenShift Container Platform with a limited set of supported features at a lower list price.
