Note: The third page is set statically to aria-current="page". If you refresh the page no matter the query parameters or hash in the URL, the third page will be set to the third page as the current page.

This is by design, as aria-current="page" will take precedence over the href in the URL.

Note: For this demo the hashchange event is triggered when the hash in the URL changes moving the aria-current="page" attribute on the correct link.

In a real world example the aria-current="page" would be moved to the correct link at page request render or application logic.
