Personalization patterns

Overview

UX personalization patterns are a way for you to quickly get started introducing targeting into your projects. They’re ready-made and contain the details you need to plug and play your concept into a personalization opportunity.

Getting started with personalization

Answer these questions

  • What audience are you trying to target?
  • How large is the audience? See this Go/No go estimator to calculate and determine viability of the project.
  • What do you want that audience to do?
  • What would success look like for this audience?
  • What type of experience do you want to use?
  • Are any tests or personalizations running on the page you’re working on?

Pattern 1: Modal

A modal personalization is an introduction of a notice that sits on top of the page and fades back all content below. This is a priority 1 implementation, as it prevents a user from interacting with the rest of the interface.

Design

Modal can be used on both desktop and mobile.

How to use

  1. Use sparingly as it interrupts the user workflow, requiring immediate attention and taking them away from their current task.
  2. Best practice is to trigger a modal when the user interacts with an element. For instance, show it if the user selects something. If that’s not possible, show it once a user has completed a task.
  3. Refrain from presenting a modal while the user is likely to be in the middle of a task.
  4. Per our design system, every modal should include a close button. It should also close when a user clicks out of the modal, and when the keyboard esc button is pressed.
  5. Messaging should be simple within a modal: don’t include multiple steps or long forms.
  6. Modal can be used on mobile, but consider that it could be harder to close a modal on a smaller screen.

The audience

A modal can be used for the following types of content (among others):

  1. Product announcements
  2. Notifications related to previous user action
  3. An experience that ideally doesn't migrate a user away from a page

Pattern 2: Sticky banner

The sticky banner experience represents a way to provide a user with a secondary offer without interrupting their experience. The banner sticks to the bottom of the screen, and overlays existing content.

Design

  1. For banners that link to a set of assets like in Pathfactory, the image shown above has been used to describe what the user gets.
  2. For banners that link to specific assets, a resource icon can be used in place of the Pathfactory image. See the XD mockup and a list of Icons for resources.
  3. The headline should not wrap. 35-42 characters or under should be safe.
  4. The supporting text should not wrap to more than 2 lines. 85-110 characters, depending on how the first row wraps, should be safe.
  5. Visitors should be able to permanently close the banner—the same banner should not pop back up upon their next visit.

How to use

  • The banner animates in from the bottom of the page. We often require that the visitor scroll past the fold before revealing the banner to avoid interfering with the page’s hero messaging.
  • Decide on which pages to show the banner. We typically do not show a banner on landing pages like the Solutions pages, and thought leadership articles.
  • Design the banner to explicitly tell the visitor what they’ll get if they click. For instance, use an image that visually describes the resource, write the description and CTA to be specific to the resource.
  • The Drift chatbot will appear in front of the banner as an icon—minimal conflict.

The audience

  • This banner can be displayed to site visitors who have shown interest in a product or topic, but have not converted.
  • We recommend the audience be large enough so that a clickthrough rate of under 10% is sufficient to meet goals.
  • Expect a fairly high close rate, based on initial trials, in the 40% range.

Pattern 3: Sticky card

The sticky card pattern is a single page or cross-page offer that persists on the side of the screen.

Design

The sticky card can hold a resource type, title and CTA. It can function in a couple of different ways depending on the primary goal of the page:

Less interruptive: on pages where the primary goal is to support reading and learning the card will fade-in to view after the visitor has scrolled 1000 pixels on the page. It will be fixed so that it follows the visitor down the page. On mobile the sticky card will be shown inline on the page. For example, we show a less interruptive card on article pages.

More interruptive: on pages where the primary goal is to present the visitor a personalized offer, the card will animate in from the right once the visitor has scrolled past the hero. On mobile the sticky card will be shown as an overlay, the same as on desktop. For example, show this on pages after a visitor has taken action that indicates they have interest in a certain subject in order to present a resource or event they may be interested in.

How to use

  • The sticky card should promote a resource or event, only allowing for a label, title and CTA.
  • This pattern can be used on a single page or across a set of pages.
  • This is useful on pages that do not have a CTA in the hero or near the top of the page as it can be used to call attention to a personalized primary resource.

The audience

  • This card can be displayed to site visitors who have shown interest in a product or topic, but have not converted.

Pattern 4: Announcement banner

The announcement banner pattern is a priority 1 personalization and should be used sparingly. This pattern places a skinny banner at the very top of the site, showing prominence over the navigation.

Design

Use the announcement banner pattern to display an important message at the top of the site. The banner can exist on one or more pages. The placement can either have a themed background image or a solid color background. To use colored backgrounds, use alert color styles.

How to use

  • If there's a logo associated with the announcement, create it at max width 80px
  • Provide a headline/description of the announcement. This text should be no longer than 60 characters
  • Provide a call to action. The text for the CTA should be around 25 characters.
  • Provide the link to the announcement if applicable.

The audience

An alert banner can be used for the following types of content (among others):

  • Important cross-site announcements: top-tier press releases, company announcements, important product launches. Use a red background.
  • Single page alerts related to a global announcement or important in-page content.

Pattern 5: In-page card

The in-page card pattern utilizes an existing card or cards within a page for personalization or targeting.

Note: cards are templated in Adobe Target only on the homepage of redhat.com. For a different card experience on other pages, new design and development work may be required.

Design

There are several versions of in-page offer cards depending on what type of asset you need to promote.

The examples are:

  • Resource offer
  • Event/webinar offer
  • Page offer
  • Product card

How to use

  1. Find an existing card or cards within your page or set of pages you want to personalize.
  2. Define the offer you’re trying to promote (event, webinar, announcement, press release).
  3. To automate the process for what you’re showing to the user, Adobe has pre-built technology to automatically personalize based on a user’s attributes or an algorithm based on content type. Please work with the Digital Systems team if you’re interested in this direction.

The audience

An in-page card should be relevant to the context of the page the user is currently on.

Pattern 6: Active tab

The active tab pattern is a simple experience applicable to any page that contains tabs. This pattern aims to customize visitor experience by highlighting a tab based on visitor’s interests.

Design

The active tab pattern uses the existing tab design, only customizing the active tab within the tabset. No custom design work is needed.

How to use

  1. Choose your target audience
  2. Choose the tab that best aligns to that audience in order to provide a targeted experience to them
  3. Request the active tab experience be set for that audience, describing what goals you hope to accomplish by doing so.

The audience

An active tab design can be used for the following types of content (among others):

  • Persona-based targeting
  • Lifecycle-based targeting
  • Product affinity targeting
  • Topic affinity targeting

Pattern 7: Inline alert

The inline alert pattern is a single or multiple page offer that appears inline with the page content.

Design

This pattern only accepts copy, no imagery. It can accommodate a single link to another location or two links. One option is to use yes | no. If “no” is selected another message will be shown:

If the visitor selects “no” consider if or when the alert should be shown again for the same visitor. It could be suppressed for a week, a month or forever depending on the goal of the project.

The alert pattern in Adobe XD has two styles per alert type. Either can be used.

How to use

  1. Use when the goal is to show a message pertinent to the surrounding content.

The audience

This card can be displayed to any appropriate audience group.

Custom patterns

Custom patterns are not pre-established targeting opportunities. If you’re interested in creating a custom targeting experience, please reach out to the digital-systems-web-team@redhat.com via this form with the details of your request and they'll help you to establish the framework for the new design.
